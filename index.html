JavaScript works in the browser using JavaScript engine, Netscape creates a browser engine for run JavaScript code, for Mozilla the engine name is Spider-Monkey, for Chrome the engine name is V8 and some of the browsers have more engines. JavaScript is a synchronous single threaded programing language. Everything in java script happens inside an Excution Context.Let's see what this Excution Context actually looks like? Excution Context is like a container.IT has two components in it, the first component is also known as the Memory Component, this is the place where all the variables and functions are stored as key and value pairs and the second component of the execution context is the Code Component.

Event Loop is very important for the JavaScript working process. When JavaScript works it has two types of things first Stack and second Queue. And it has one more section that's called the heap. It's all working like a loop and the loop process name is Event Loop. The moral of the story is when Javascript executes overall, the browser means the V8 engine executes it but it all has been stored in the browser, the browser manages it, and when what will work, the browser manages it. And the whole process name is Event Loop.

The browser has two different storages. Session Storage and Local Storage. The storage stores two types of data, it's called Key and Value. And every single value is stored as an object. But the main difference between Local Storage and Session Storage is Local Storage stores data for permanent and Session Storage site data for temporary. As an example, if you open a browser and browse something then close the browser when you close the browser the session storage data was removed but the local storage still stores those data and when you again the browser works for that depend on past data. That's the main difference between Session Storage and Local Storage.
